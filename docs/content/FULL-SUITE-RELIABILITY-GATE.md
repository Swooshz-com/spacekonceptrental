# Full-Suite Reliability Gate

This Phase 4D-A/B reliability gate is repo-local, template-only, non-live, and not evidence. It defines expected local validation behavior without recording actual run results, preview evidence, production evidence, owner sign-off, provider approval, or deployment approval.

Full website tests must not hang. Targeted tests do not replace the full suite. CI-green is required before merge. Docker unavailability may be reported in PR text only. No validator or suite may be altered to skip Docker-required checks. No safety assertion may be removed just to pass tests.

| Command | Expected pass condition | Blocked condition | Allowed environment limitation | Forbidden workaround | Evidence boundary |
| --- | --- | --- | --- | --- | --- |
| `cd website && npm test` | Full Vitest suite completes deterministically without `.skip`, `.only`, fake pass placeholders, or weakened safety coverage. | Hang, open handle, skipped meaningful test, or removed leakage/fake-fact/provider/deployment assertion. | Local machine resource variance may be described in PR text only. | Do not replace the full suite with targeted tests only. | [NOT EVIDENCE / NOT RECORDED] |
| `cd website && npm run typecheck` | TypeScript completes with no type errors. | Type error, missing type coverage, or runtime boundary weakened to satisfy types. | None expected for repo-local typecheck. | Do not loosen types around admin, provider, or public safety boundaries. | [NOT EVIDENCE / NOT RECORDED] |
| `cd website && npm run build` | Next build completes locally without deployment, provider connection, or live preview smoke. | Build failure or new env/provider dependency. | Build-time host resource variance may be described in PR text only. | Do not add provider config, deployment config, or fake env values. | [NOT EVIDENCE / NOT RECORDED] |
| `npm run validate:owner-review-rehearsal` | Phase 4C rehearsal docs and Phase 4D roll-forward boundaries remain safe. | Filled owner-review evidence, public leakage, fake facts, provider scope, deployment scope, or ecommerce wording. | None expected for repo-local validator. | Do not remove safety assertions to pass. | [NOT EVIDENCE / NOT RECORDED] |
| `npm run validate:preview-handoff` | Preview handoff remains local-only and blocked from provider/deployment actions. | Filled preview evidence, production evidence, provider config, live smoke, or deployment approval claim. | None expected for repo-local validator. | Do not convert approval placeholders into evidence. | [NOT EVIDENCE / NOT RECORDED] |
| `npm run validate:local-release-candidate` | Local release-candidate package remains deterministic and safety-gated. | Public internals leak, fake fact appears, forbidden runtime/provider file appears, or suite bypass is added. | None expected for repo-local validator. | Do not weaken release-candidate or public leakage assertions. | [NOT EVIDENCE / NOT RECORDED] |
| `npm run validate:local-freeze` | Phase 4D freeze docs, admin snapshot, tests, and roll-forward status are present and safe. | Missing Phase 4D doc, missing script wiring, filled evidence, public leak, Docker-skip bypass, `.skip`, or `.only`. | None expected for repo-local validator. | Do not add broad bypasses or placeholder pass tests. | [NOT EVIDENCE / NOT RECORDED] |
| `npm run validate:release-candidate-suite` where Docker/Supabase is available | Full local suite passes, including Docker/Supabase-required checks, without deployment. | Docker-required checks fail where Docker/Supabase is available, or suite is weakened. | Docker/Supabase unavailable locally may be reported in PR text only. | Do not add Docker-skip bypass logic or mark Docker checks optional inside the suite. | [NOT EVIDENCE / NOT RECORDED] |
