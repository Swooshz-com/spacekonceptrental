# Quote/Enquiry Public Expectation Boundary

This document is repo-local, template-only, non-live, and not evidence. It defines public-safe quote/enquiry behaviour for the current furniture/event rental website. It does not record owner acceptance, user testing, preview evidence, production evidence, or owner sign-off.

Deployment approval remains not granted.

## Public-safe behaviour

- The public quote form is an enquiry/request intake, not a confirmed booking.
- Public confirmation is receipt-style only.
- Public users must not see admin status, internal notes, recovery statuses, destructive-action safeguards, or admin URLs.
- There is no public tracking lane.
- There is no customer account.
- There is no payment/checkout/order language.
- There are no notification/CRM/runtime promises.
- There is no guaranteed response time unless the owner supplies it later and a separate approved implementation supports it.

## Safe public copy examples

- "Send a rental quote enquiry with your event date, venue or location, requested listings or items, quantities, alternates, and setup notes."
- "Your submission was received as an enquiry. The team can review the details and follow up directly."
- "Selected listings are a starting point for the rental request and do not confirm availability, fit, or final quote details."
- "Use listing names or item descriptions if you are still comparing event furniture rental options."

## Unsafe public copy examples

- "Your booking is confirmed."
- "Availability is guaranteed."
- "Track your quote status online."
- "Create a customer account to manage the request."
- "Upload your own images and manage the setup yourself."
- "Pay now, checkout, or complete your order."
- "A notification or CRM workflow will send automatic updates."
- "The team will respond within [UNSUPPLIED RESPONSE TIME]."
