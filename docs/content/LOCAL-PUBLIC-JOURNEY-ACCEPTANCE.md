## Public browse path

Public visitors can start from the homepage, browse listings, compare categories, review event-use guidance, open a rental listing, and send a quote/enquiry request. Public CTAs stay limited to Browse listings, View rental listing, Request a quote, Send an enquiry, and Start a rental enquiry.

## Catalogue, listing, category, and event continuity

- Catalogue/listing index: public rental/event furniture listing cards link to listing details and quote/enquiry intake.
- Category routes: category cards link to filtered listings and safe enquiry recovery.
- Listing detail routes: detail pages treat the listing as browsing context and pass selected listing context into the request form.
- Event-use routes: event setup guidance helps visitors describe the request without implying a fixed package, availability, or final rental outcome.

## Listing-to-enquiry continuity

Selected listing context is a starting point only. It can prefill or display editable request text in the quote/enquiry form, but it does not confirm rental fit, set aside furniture, create a customer account, create public tracking, upload files, send notifications, create CRM work, or create a transaction.

## Quote/enquiry receipt boundary

Quote/enquiry submission copy may say the request was received, will be reviewed, and may be followed up directly. Public references remain receipt/reference only. The public flow must not promise availability, response time, final rental details, provider setup, deployment status, or owner approval.

## Public fallback and not-found safety

Public empty states, missing category/listing states, and generic not-found recovery should guide visitors back to browsing listings, categories, event guidance, or quote/enquiry intake. Public recovery must not expose admin URLs, owner handoff internals, release-control details, internal notes, destructive-action safeguards, recovery lanes, or status-transition matrix details.
