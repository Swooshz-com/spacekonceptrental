# Owner-Demo Issue Backlog

This backlog is repo-local, template-only, and non-live.

It captures possible follow-up issue shapes for an owner-demo pass without
recording real owner corrections, owner sign-off, filled evidence, deployment
approval, provider configuration, or production facts. Keep every entry as a
placeholder until an explicitly approved future owner-review correction PR uses
real owner input.

## Status And Severity Options

Use these labels without implying launch approval:

- Product polish.
- Owner input required.
- Blocks owner review.
- Blocks future launch/deployment.
- Deferred after launch.
- Not in current scope.

Safe status field:

```text
[STATUS: OPEN / OWNER INPUT REQUIRED / LOCALLY RESOLVED / DEFERRED]
```

Deployment boundary field:

```text
[DEPLOYMENT APPROVAL: NOT GRANTED]
```
